## 2024-05-18 - [Optimizing Arrays inside Renders and Precompiling RegExp]
**Learning:** In a highly dynamic, list-rendering component like `Dashboard.jsx`, creating `RegExp` objects inside a map function during render (as was happening with `.replace(new RegExp(...))` in `normalizeNameAlt`) causes a massive amount of unnecessary object instantiations and garbage collection pressure. Additionally, using `Array.prototype.includes()` inside `.sort()`, `.filter()`, and `.map()` iterations leads to O(N^2) or worse complexity and degrades UI performance.
**Action:** Hoist repetitive `RegExp` creations outside the component body (e.g. `const STOP_WORDS_REGEX = new RegExp(...)`) to precompile them once. Convert lookup arrays (like `disabledIngredients`) to `Set` instances using `useMemo` to ensure O(1) lookups during array operations in React renders.
