## 2024-05-19 - [O(1) Data Structures and Precompiled Regexes]
**Learning:** React component iteration speed heavily suffers when looping string regex patterns or making `Array.includes` O(N) array look-ups on large lists within tight loops like `.filter()`. We discovered a tight loop inside `Dashboard.jsx` iterating over a 39-item Array and instantiating 39 new `RegExp` objects per-invocation to normalize strings, and an $O(N)$ `includes` on a state array used to filter and sort thousands of ingredients.
**Action:** Always precompile long static dictionaries (like stop-words) into a global outer component scope as a single massive OR statement regex `new RegExp('\b(a|b|c)\b', 'gi')`. Next, if a React state array is used heavily as a membership filter (`if list.includes()`), memoize it instantly into a Set (`useMemo(() => new Set(arr))`) to reduce Big O lookup time down to O(1).
