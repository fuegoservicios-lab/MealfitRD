## 2024-05-24 - Pre-compiled Regex and Hoisted Constants for Performance
**Learning:** Found a performance bottleneck in `Dashboard.jsx`'s `normalizeNameAlt` where a static `stops` array was being converted to 39 separate RegExp objects via a `for...of` loop *on every function invocation*, and the `irregulars` dictionary was re-allocated on every iteration of a `.map` loop within that function.
**Action:** When implementing text normalization utilities in React components that are called repeatedly (e.g. for every item in an array), hoist static configuration arrays, dictionaries, and specially, aggregate multiple static word replacements into a single pre-compiled `RegExp` (e.g. `new RegExp('\\b(' + STOPS.join('|') + ')\\b', 'gi')`) outside the component to change the time complexity from O(N*M) back to O(N) and reduce excessive garbage collection.
